## 🌐 النشر على Railway

### المتطلبات:
- Node.js >= 20.9.0
- Railway account
- GitHub repository

### الإعدادات:

**package.json**:
```json
{
  "engines": {
    "node": ">=20.9.0"
  },
  "scripts": {
    "build": "next build",
    "start": "next start"
  }
}
```

**nixpacks.toml**:
```toml
[phases.setup]
nixPkgs = ["nodejs_20"]
```

**railway.toml**:
```toml
[build]
builder = "NIXPACKS"
buildCommand = "npm run build"

[deploy]
startCommand = "npm start"

[env]
NODE_ENV = "production"
NODE_VERSION = "20.9.0"
```

### خطوات النشر:
1. ربط GitHub repository مع Railway
2. Railway يبني المشروع تلقائياً
3. انتظر 2-3 دقائق
4. التطبيق جاهز!

## 🔧 استكشاف الأخطاء

### المشكلة: الصور لا تظهر على Railway

**الحل**:
1. تحقق من Build Logs على Railway
2. افتح `/api/check-images` للتحقق من وجود الصور
3. افتح `/test-images` لاختبار جميع الصور
4. امسح Cache المتصفح (Ctrl + Shift + Delete)

### المشكلة: Build فشل على Railway

**السبب المحتمل**: Node.js version قديم

**الحل**:
- تأكد من `"engines": { "node": ">=20.9.0" }` في package.json
- تأكد من وجود `nixpacks.toml`

### المشكلة: مهندس يدخل على منطقة خاطئة

**السبب**: localStorage يحتوي على صلاحيات قديمة

**الحل**:
```javascript
// في Console (F12)
localStorage.clear()
// ثم سجل دخول مرة أخرى
```

## 🔐 نظام الصلاحيات

### كيف يعمل:

1. **عند تسجيل الدخول**:
   - يحفظ بيانات المستخدم في localStorage
   - إذا كان له areaId، يحفظ صلاحية منطقته فقط
   - إذا كان يوزر خاص (gm, QTY, QTY2)، يحفظ صلاحيات جميع المناطق

2. **عند الدخول على المكتب الفني**:
   - يتحقق من localStorage للصلاحيات
   - يسمح بالدخول فقط للمناطق المصرح بها

3. **عند الدخول على الدليل**:
   - جميع المستخدمين يمكنهم الدخول (لا قيود)

### الملفات المهمة:
- `app/page.tsx` - تسجيل الدخول وحفظ الصلاحيات
- `app/technical-office/login/page.tsx` - التحقق من صلاحيات المناطق
